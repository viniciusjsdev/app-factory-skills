# Feature Definition of Done

- [ ] Acceptance criteria and non-goals are explicit
- [ ] Route is thin and feature ownership is clear
- [ ] Visible interactions are functional
- [ ] Loading, empty, error, success, disabled, and permission states are covered
- [ ] UI does not access mocks, API, storage, or environment directly
- [ ] Mobile and keyboard behavior are usable
- [ ] Critical view-model, validation, and mapping behavior is tested
- [ ] API and architecture documentation is current
- [ ] Required validation commands passed or blockers are recorded
