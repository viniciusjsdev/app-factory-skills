# Create Feature Workflow

1. Define the user outcome, route, permissions, states, and acceptance criteria.
2. Create only the required feature folders: page, components, hooks, services, repository, schema, and types.
3. Keep the route thin and lazy-load the feature entry.
4. Build the view model before scattering state across components.
5. Hide mock, storage, and API details behind the repository contract.
6. Implement all visible actions and feedback states.
7. Test critical behavior and data mapping.
8. Update API and architecture documentation when the boundary changes.
