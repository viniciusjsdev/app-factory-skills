# Backend Validation

- [ ] ORM classes use CamelCase.
- [ ] Model specifications live in configurations.
- [ ] ORM access exists only in repositories.
- [ ] Services own business rules without database or HTTP access.
- [ ] Controllers use DTO-defined payloads and contain endpoint transport only.
- [ ] Migrations were generated by Django commands.
- [ ] Tests and architecture scan passed or limitations are documented.
