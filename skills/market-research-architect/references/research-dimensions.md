# Research dimensions

Select only dimensions that can change the target decision.

## Core dimensions

| Dimension | Questions | Typical evidence |
|---|---|---|
| Problem | Is the problem frequent, costly, urgent, or risky? | User language, workflows, complaints, studies, direct experiments |
| Audience | Who experiences, uses, buys, approves, or benefits? | Segment data, communities, job roles, platform-native research |
| Alternatives | What is used today, including manual work and doing nothing? | Competitors, substitutes, reviews, process evidence |
| Demand | Does behavior indicate interest or willingness to act? | Search/category signals, waitlists, conversations, paid-intent tests |
| Economics | Can value, price, acquisition, and delivery plausibly coexist? | Pricing, app estimates, unit-economics scenarios, comparable businesses |
| Distribution | Can the audience be reached in a repeatable way? | Channels, creators, communities, keywords, sales paths |
| Timing | Why now, and can the opportunity persist? | Trend timelines, policy/technology changes, cohort momentum |
| Feasibility/risk | What technical, legal, trust, or operational constraint can invalidate it? | Primary documentation, scientific evidence, regulations, experiments |

## Selection rule

For each selected dimension, name the decision it informs and the falsifier. If a dimension cannot change the decision, remove it from the current research contract.

## Common mistakes

- treating a large category as evidence for the proposed niche;
- defining the customer only by demographics rather than behavior and context;
- counting competitors as demand without measuring their performance or relevance;
- using downloads as active users, revenue as profit, or ratings as satisfaction;
- using academic plausibility as commercial demand;
- using platform popularity without checking audience fit and reachable distribution.
