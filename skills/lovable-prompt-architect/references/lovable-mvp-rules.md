# Lovable MVP Rules

## First Prototype

The first Lovable prototype should usually avoid real backend.

Prefer:

- mock data
- localStorage
- fake loading
- simulated services
- realistic interactions
- frontend-only flows

## Negative Scope

Always clarify what not to implement.

Examples:

- no real authentication
- no real API
- no real payment
- no real OCR
- no real AI inference
- no real database
- no real file processing

## Why

This keeps Lovable focused on building a convincing visual prototype instead of getting stuck on integrations.
