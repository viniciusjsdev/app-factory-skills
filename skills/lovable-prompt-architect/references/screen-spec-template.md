# Screen Spec Template

For each screen, define:

## Route

Example: `/dashboard`

## Goal

What the screen is supposed to help the user do.

## Layout

How the screen is visually organized.

## Components

Cards, tables, forms, charts, lists, modals, drawers, upload zones, etc.

## Data

What mock data appears.

## Actions

What buttons and interactions exist.

## Feedback

Loading, toast, modal, disabled state, success, error, empty state.

## Mobile Behavior

How the screen adapts to smaller viewports.
