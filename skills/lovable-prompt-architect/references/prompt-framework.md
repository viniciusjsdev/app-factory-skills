# Lovable Prompt Framework

A strong Lovable prompt has these layers:

1. Product definition
2. MVP boundaries
3. Target users and pains
4. Stack
5. Visual identity
6. UX rules
7. Routes
8. Layout
9. Screen-by-screen specification
10. Mock data
11. Mock services
12. Interactions
13. Responsiveness
14. Quality expectations
15. First delivery order

## Rule

The prompt must be specific enough to reduce ambiguity in Lovable.

Do not write only feature names. Describe what each feature must look like, how it behaves and what mock data it uses.
