# Manus credit policy

## Minimize paid-agent work

Use Codex for reasoning, discovery synthesis, copy, creative production, formatting, validation, and troubleshooting. Send Manus a final, bounded operation only when native platform access or a connected action is the unique requirement.

## Task shape

- one operation type per task;
- exact connected account and artifact version;
- bounded item count, retries, and elapsed time;
- no open-ended browsing or strategy;
- no request to improve/rewrite content;
- no recipient discovery inside a contact task;
- explicit expected receipt fields.

Batch only homogeneous actions with the same approval and risk profile. Do not batch merely to reduce apparent task count.

## Accounting

Record provider task ID, reported credits/usage when available, started/completed times, retries, and outcome. Provider pricing can change; do not promise a maximum credit cost unless the current provider API contract enforces it.

## Escalation

If a task needs wider discovery, more recipients, a new channel, a content rewrite, additional permissions, or extra spend, stop and return to Codex/human approval rather than consuming credits autonomously.
